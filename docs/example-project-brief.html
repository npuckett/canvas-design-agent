Project Description

Design and build an interactive installation that responds to human presence, movement, or input. Your installation should demonstrate a clear feedback loop and consider the relationship between physical space and digital response.

You may work individually or in pairs. Pair projects are expected to be proportionally more ambitious in scope.

Learning Objectives

By completing this project, you will demonstrate your ability to:

  1. Design an interactive system with clear input/output relationships
  2. Implement real-time sensor-based interaction using physical computing
  3. Apply feedback loop principles to create meaningful user experiences
  4. Document your design process from concept to completion
  5. Present and critique interactive work in a studio context

Detailed Rubric
Concept & Design (25%)
Excellent: Original concept with clear artistic intent. Strong connection between concept and interaction design. Thoughtful consideration of spatial context.
Needs improvement: Concept is underdeveloped or derivative. Weak connection between idea and implementation.
Technical Execution (30%)
Excellent: Reliable, responsive system. Clean code. Hardware and software work seamlessly. Handles edge cases.
Needs improvement: Frequent crashes or unresponsive behavior. Code is disorganized. System fails under normal use.
User Experience & Feedback Loop (20%)
Excellent: Clear, intuitive interaction. Users understand the system without instruction. Feedback loop is evident and well-calibrated.
Needs improvement: Users are confused about how to interact. Feedback is delayed, absent, or unclear.
Documentation & Process (15%)
Excellent: Thorough process blog showing iterative development. High-quality video and photos. Thoughtful reflection essay.
Needs improvement: Minimal documentation. Missing blog posts or video. Reflection is superficial.
Presentation (10%)
Excellent: Clear, well-rehearsed presentation. Live demo works. Engages with questions thoughtfully.
Needs improvement: Unprepared, demo fails, unable to discuss design decisions.
Technical Requirements & Resources
Note: All hardware is available for sign-out from the Equipment Room (EV 5.601). Reserve at least one week in advance.

Minimum Technical Requirements:

  • At least one physical sensor or input device
  • Real-time processing (latency under 100ms)
  • At least one output modality (visual, audio, or haptic)
  • Must function as a standalone installation (no laptop screen as primary display)

Available Equipment:

  • Arduino Uno / Mega boards
  • Raspberry Pi 4 units
  • Kinect v2 depth sensors
  • Ultrasonic distance sensors, PIR motion sensors, pressure mats
  • LED strips (NeoPixel), projectors, speakers
