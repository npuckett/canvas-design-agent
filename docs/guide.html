Getting Started

  1. Get the skill file. Download or clone SKILL.md from the GitHub repository. This single file contains all Canvas constraints and element templates.
  2. Browse the Element Catalog. Look through available elements and note any you want to use. Each element has an ID (e.g., V05, C01) you can reference.
  3. Write your content. Describe your page in plain text -- schedules, assignments, resources, anything. Optionally mention element IDs.
  4. Choose a workflow. Use a local agent (VS Code, Cursor) or a web-based agent (ChatGPT, Claude, Gemini). See below.
  5. Paste into Canvas. Open your Canvas page, switch to the HTML editor (</> icon in RCE), paste the generated HTML, and save.

First time? Try the walkthrough below -- it takes about 5 minutes and requires zero HTML knowledge.

Your First Page (5-Minute Walkthrough)

This walkthrough creates a real Canvas page from plain text. No HTML knowledge needed -- you just describe what you want and the AI does the rest.

  1. Write your content in plain text. Open any text editor (even Notepad) and type something like this:
    # Simple Syllabus Page Transform this into a Canvas page using the skill. Pick whatever layout and design elements look best. ## Course Info ART 101 — Introduction to Drawing Fall 2026 Tuesdays 1:00–4:00 PM, Room 302 ## About This Course This studio course introduces fundamental drawing concepts including line, form, value, and composition. ## Weekly Schedule Week 1 — Intro & Materials Overview Week 2 — Line and Contour Drawing Week 3 — Value Studies Week 4 — Form and Volume ## Grading Sketchbook: 30% Studio Projects: 40% Midterm Critique: 10% Final Critique: 10% Participation: 10%

    Notice there are no element IDs or HTML -- just plain text with headings. This is all you need to get started. (See simple-syllabus.md for a more complete version.)

  2. Give it to an AI agent along with SKILL.md.
    • Web workflow (easiest): Open ChatGPT, Claude, or Gemini. Upload or paste the contents of SKILL.md, then paste your content and say "Transform this into a Canvas page using the skill."
    • Local workflow: If you use VS Code with Copilot, place SKILL.md in your project's .github/ folder, save your text as a .md file, and ask Copilot to transform it.
  3. Paste the HTML into Canvas. The agent returns HTML code. To put it in Canvas:
    1. Open your Canvas course and navigate to the page you want to edit (or create a new one).
    2. Click Edit.
    3. In the Rich Content Editor toolbar, click the HTML editor icon -- it looks like </> and is usually in the toolbar's overflow menu (click ⋮ or more options if you don't see it).
    4. Select all existing content in the HTML view and delete it.
    5. Paste the generated HTML.
    6. Click Save (or Save & Publish).

What you'll get: A professionally formatted page with a colored header, organized schedule table, grading breakdown, and consistent styling -- all from plain text input. Every style is inline, so Canvas won't strip anything.

Workflow A: Local Agent

Best for faculty using a code editor (VS Code with Copilot, Cursor, Windsurf) who want to manage multiple pages in a project.

  1. Set up the skill. Copy SKILL.md into your project's .github/ directory. Most AI-enabled editors automatically read instruction files from this location.
  2. Create a prompt file. Write your page content in a .md file. Use plain language -- describe what the page should contain, its structure, and reference any element IDs you want (see Prompt Files).
  3. Ask the agent to generate HTML. In your editor's AI chat, ask it to convert your prompt file into a Canvas HTML file. Example: "Transform this into a Canvas HTML file using the skill. Use a gradient header and collapsible sections."
  4. Review the HTML file. The agent writes an .html file in your project. You can preview it in a browser, keep it under version control, and regenerate it any time your content changes.
  5. Paste into Canvas. Open the generated HTML file, copy its contents, switch to the HTML editor view in Canvas RCE, paste, and save.

Tip: Keep your prompt files and generated HTML organized by course. A typical structure is one folder per course with .md prompt files alongside their generated .html output files.

Workflow B: Web-Based Agent

Best for quick, one-off page generation without installing anything. Works with ChatGPT, Claude, Gemini, or any web AI tool.

  1. Upload the skill. Start a new conversation. Upload or paste the contents of SKILL.md. This gives the agent all Canvas constraints and element templates it needs.
  2. Provide your content. Type or paste your page content in the same conversation. Describe what kind of page it is and what elements you want.
  3. Get the HTML. The agent generates Canvas-ready HTML. Copy the entire output (or download it as an .html file if the tool supports it).
  4. Save the HTML file. Save the generated HTML to your project so you can reference, revise, and reuse it later.
  5. Paste into Canvas. Open the HTML file, copy its contents into Canvas RCE's HTML editor view, and save.

Tip: If you have a course-specific template, upload it alongside SKILL.md so the agent applies your course's colors and layout consistently.

Writing Prompt Files

A prompt file is a plain markdown file that describes your Canvas page. You write your content naturally, reference element IDs where you want specific formatting, and let the agent generate an HTML file. The generated HTML file is your reusable output -- keep it in your project, preview it in a browser, revise it by editing the prompt and regenerating, and paste it into Canvas when ready.

Anatomy of a Prompt File

A prompt file has three parts:

  1. Instruction block -- A blockquote at the top telling the agent what to do and which elements to use.
  2. Content sections -- Your actual page content organized with markdown headings.
  3. Element hints -- Parenthetical notes like "(collapsible)" or "(use D07 progress bars)" guiding the agent's choices.

Minimal Example

# Week 3 Class Page > Transform this into a Canvas page using the skill. > Use V02 blue header, C06 agenda list, C01 collapsible for resources. ## Header Class 03: Sensor Fundamentals CART 310 · Week 3 · September 22, 2026 ## Agenda 1. Review: Input/Output exercise (10 min) 2. Lecture: Sensor types and applications (40 min) 3. Break (10 min) 4. Lab: Sensor workshop (60 min) ## Resources (collapsible) - Slides: sensor-fundamentals.pdf - Arduino sensor starter kit docs - Recommended: Igoe, "Making Things Talk" Ch. 4

The agent reads this file plus SKILL.md and outputs complete Canvas-ready HTML with inline styles.

Tips for Good Prompt Files

  • Reference elements by ID -- Write "V05 gradient header" or "D01 data table" so the agent uses the exact element you want.
  • Use markdown structure -- Headings (##) become page sections. Lists become formatted lists. Tables become data tables.
  • Add hints in parentheses -- Write "(collapsible)" after a heading to get C01, or "(3 cards in a row)" to get L03 flexbox.
  • Keep it natural -- You don't need to know HTML. Write like you're describing the page to a colleague.
  • One file per page -- Each prompt file produces one Canvas page. Keep them organized by course or week.

Prompt File Conventions

For best results across different AI tools, follow these formatting patterns:

Pattern When to Use Example
## Section Name (Element ID) When you want a specific element ## Quick Info (L03)
## Section Name When you want the agent to choose ## About This Course
## Section Name (hint) When you want a behavior but don't know the ID ## Resources (collapsible)

All three patterns work. Element IDs are optional but give you more precise control. As you get familiar with the catalog, you'll naturally start using IDs for the elements you like.

Course-Specific Templates

A course-specific template combines element library blocks into a consistent design you reuse across an entire course. Define your visual identity once and every page automatically follows it.

Using a Template

With a Local Agent

  1. Save your template file in the project alongside SKILL.md.
  2. Reference both when prompting: "Using the course template in course-template.txt, transform today's class notes into a Canvas page."

With a Web Agent

  1. Upload both SKILL.md and your template file at the start of the conversation.
  2. Provide your content and ask for a Canvas page. The agent applies both constraints and template preferences.

Building a Template Library

  • Course template -- Per-course design (colors, headers, standard sections)
  • Assignment template -- Objectives, requirements, rubric, submission instructions
  • Syllabus template -- Course outline with collapsible policy sections
  • Schedule template -- Course calendar as D05 grid with color-coded rows
  • Project brief template -- Description, milestones, deliverables, grading

Iterating on Templates

  1. Start simple. Define colors and a header style. Generate a few pages.
  2. Review in Canvas. Paste the HTML and see how it looks.
  3. Refine. Adjust template preferences based on what works.
  4. Expand. Add page-type variations and section templates.
  5. Reuse. Next semester, update dates and topics -- template stays the same.

Canvas Constraints Quick Reference

The SKILL.md contains the full constraint list. The agent handles these automatically, but here's a summary.

Works in Canvas

  • All inline style="" attributes
  • Flexbox and CSS Grid
  • Linear gradients
  • Relative and absolute positioning
  • <details>/<summary> collapsible sections
  • <mark>, <abbr>, <ins>, <del>, <kbd>
  • All table features: colspan, rowspan, colgroup, caption, thead/tbody/tfoot
  • Audio and video players
  • Definition lists (<dl>)

Stripped by Canvas

  • <style> blocks, <script> tags, external CSS/JS
  • All SVG elements
  • <meter>, <progress>, <fieldset>, <legend>
  • box-shadow, text-shadow, opacity, transform
  • letter-spacing, word-spacing
  • data: URIs in image sources
  • The open attribute on <details>
  • The reversed attribute on <ol>

Troubleshooting

Common issues when working with Canvas HTML and how to fix them.

My formatting disappeared after saving

Canvas strips <style> blocks, <script> tags, and certain CSS properties when you save. If your formatting vanished, the HTML likely used one of these stripped features. Regenerate the page and tell the agent: "Make sure all styles are inline -- no <style> blocks." The SKILL.md file prevents this, but if you're editing HTML manually, keep all styles in style="" attributes.

Images aren't showing

Canvas cannot resolve relative image paths like images/photo.jpg. Every image URL must be absolute:

  • Canvas-hosted: Upload to Files in your Canvas course, then use the full Canvas file URL.
  • GitHub Pages: Use the complete https://username.github.io/repo/images/photo.jpg URL.
  • External: Use any publicly accessible https:// URL from a domain Canvas allows.

If you see a broken image icon, right-click it and check the URL -- it's almost always a relative path or a blocked domain.

Collapsible sections won't stay open

Canvas strips the open attribute from <details> elements. All collapsible sections start closed -- this is a Canvas limitation and cannot be overridden. Design your page so that critical information is visible by default (outside collapsibles) and supplementary content is collapsed.

Layout looks wrong on mobile

Multi-column layouts (flexbox, grid, tables) may not wrap well on small screens. To improve mobile layout:

  • Use flex-wrap: wrap and min-width: 200px on flex children (the L03 template includes this).
  • Avoid fixed-width columns -- use percentages or fr units.
  • Test by resizing your browser window before pasting into Canvas.

Shadows, animations, or effects aren't working

Canvas strips box-shadow, text-shadow, opacity, transform, and all JavaScript. These cannot be used. For visual depth, use borders, background colors, and gradients instead. The element catalog is designed around what Canvas actually supports.

The agent generated a full HTML document

If the output includes <html>, <head>, or <body> tags, only copy the content inside the <body> tag. Canvas expects an HTML fragment, not a full document. Tell the agent: "Output only the HTML fragment -- no html, head, or body tags."

Colors or fonts look different in Canvas

Canvas may apply its own base styles on top of your inline styles. If colors seem off, make sure every text element has an explicit color in its inline style. The design system defaults in SKILL.md are chosen to work well with Canvas's base styles.
